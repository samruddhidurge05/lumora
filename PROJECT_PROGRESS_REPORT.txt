OVERALL ARCHITECTURE
--------------------
Split-Architecture / Dual-Data approach:

1. Frontend (React/Vite) at localhost:5173
   - Auth: entirely Firebase-driven via AuthContext.jsx
   - Real-time data (chat, messaging): Firebase Firestore
   - Backend API calls via backendFetch (utils/api.js) to localhost:8000/api
   - Auth token (lumora_backend_token) stored in localStorage

2. Backend (FastAPI/Python) at localhost:8000
   - Full REST API at /api/* with Swagger docs at /docs
   - SQLite-backed with 18 SQLAlchemy table models
   - JWT-based authentication separate from Firebase
   - 16 routers mounted; many sub-modules are stubs

ARCHITECTURAL NOTE (RESOLVED):
  The Firebase-to-Backend JWT auth bridge has been implemented as of 2026-06-29.
  POST /api/auth/firebase-sync now exchanges Firebase ID Tokens for backend JWTs.
  Frontend stores lumora_backend_token and lumora_backend_uid in localStorage.
  All backend API calls from logged-in users are now correctly authenticated.

  Role Storage: localStorage as lumora_active_role
  Protected Routes: ProtectedRoute checks login + role match
  BRIDGE CALL (NEW): syncWithBackend() called in onAuthStateChanged after role
    resolution. On logout: clearBackendToken() removes lumora_backend_token
    and lumora_backend_uid from localStorage.

BACKEND AUTH (JWT - auth_router.py + dependencies.py):
  POST /api/auth/register     -> creates User in SQLite with hashed password (WORKS)
  POST /api/auth/login        -> verifies hash, returns JWT {"sub": user.id} (WORKS)
  GET  /api/auth/me           -> returns current user from JWT (WORKS)
  POST /api/auth/firebase-sync -> NEW: verifies Firebase RS256 token, upserts user,
                                  returns HS256 JWT + user.id for localStorage
  get_current_vendor: ✔ Mock fallback REMOVED. Real JWT always required.
  get_current_user_required:  ✔ Correctly raises 401 when no/invalid token.
  Token flow: lumora_backend_token (JWT) + lumora_backend_uid (str(user.id))
              both stored in localStorage after firebase-sync.

FIREBASE-TO-BACKEND BRIDGE (IMPLEMENTED 2026-06-29):
  Flow:
    1. User logs in via Firebase (any provider)
    2. onAuthStateChanged fires -> AuthContext calls syncWithBackend(firebaseUser)
    3. authService.js calls getIdToken() -> POST /api/auth/firebase-sync
    4. Backend verifies RS256 Firebase ID Token using Google public keys
    5. Backend upserts User in SQLite (by email); returns HS256 JWT
    6. Frontend stores: lumora_backend_token (JWT) and lumora_backend_uid (str(user.id))
    7. All backendFetch() calls attach Authorization: Bearer <token>
    8. On 401: auto-refresh Firebase token -> re-sync -> retry (utils/api.js)
    9. On logout: clearBackendToken() removes both localStorage keys
  Status: FULLY FUNCTIONAL. All 10 security tests pass.

PASSWORD RESET:
  Backend: /api/auth/forgot-password returns canned message, no email sent
  Frontend: Uses Firebase sendPasswordResetEmail() (works correctly client-side)

EMAIL VERIFICATION:
